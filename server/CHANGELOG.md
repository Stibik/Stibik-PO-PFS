# История версий

Правило на будущее: при каждом обновлении — новая запись сверху + бампаем
`"version"` в `package.json`. Версия видна в самом приложении (подвал страницы)
и через `GET /api/meta/version`.

## v0.4.0 — фильтры, колонки и детали заказа Kaspi
- **Статус заказа** теперь показывается понятным словом (например "В архиве" вместо "ARCHIVE") —
  но это по-прежнему предположение по названиям; уточним дальше по кнопке «Детали»
- **Фильтры по статусу** — кнопки над таблицей заказов Kaspi, с количеством в каждой категории
- **Настройка колонок** — галочки, что показывать в таблице (можно скрыть ненужное)
- **Кнопка «Детали»** на каждой строке — показывает вообще все поля, которые реально прислал
  Kaspi по этому заказу. Нужно, чтобы найти правильные названия полей для наименования товара
  и даты передачи курьеру (сейчас показывают "—", т.к. предполагаемые названия полей не совпали
  с реальными — посмотрим вместе через «Детали» и поправим маппинг точно)

## v0.3.1 — исправлен лимит дат Kaspi
- Kaspi ограничивает диапазон фильтра по дате максимум **14 днями** (не 30, как было изначально
  предположено) — реальный API ответил на это ошибкой 400, теперь исправлено на 14
- Пока синхронизация подтягивает только последние 14 дней. Если нужно поднять более старые
  заказы — понадобится доработка (несколько запросов окнами по 14 дней подряд), могу добавить
  отдельным шагом

## v0.3.0 — исправлена синхронизация Kaspi + фото + экспорт в Excel
- **Исправлена ошибка синхронизации с Kaspi**: реальный API потребовал обязательный
  фильтр по дате (`creationDate`), которого не было в первой версии.
- **Фото добавлены в «Приход» и «Прайс»** — при создании/редактировании записи можно
  прикрепить фото, оно сохраняется на сервере и отображается в таблице
- **Экспорт в Excel** — кнопка в «Приходе» и в «Прайсе», выгружает текущий список в .xlsx

## v0.2.0 — Приход / Заказы (Kaspi) разделены
- Вкладка «Заказы» переименована в **«Приход»** — это ручной учёт (в пути/пришло/проблема),
  как и раньше. Каждой записи теперь автоматически присваивается уникальный номер (`receiptNumber`)
- Новая вкладка **«Заказы»** — сюда попадают только заказы, подтянутые из Kaspi по API.
  У каждого — свой уникальный номер (`displayNumber`), код заказа Kaspi, магазин-источник,
  наименование (если Kaspi его отдаёт) и дата передачи курьеру
- Токены API для 3 магазинов Kaspi теперь можно вводить **прямо в интерфейсе**
  (вкладка «Kaspi: настройки»), не только через переменные окружения на сервере.
  Сохранённый токен показывается в замаскированном виде (не как полноценное шифрование,
  а просто чтобы не светить его на экране лишний раз)
- Данные обоих типов заказов физически хранятся в одном списке с полем `source`
  (`manual` / `kaspi`) — так удобнее синхронизировать и фильтровать

## v0.1.0 — первая онлайн-версия (фундамент)
- Сервер на Node.js/Express, вход по логину/паролю (без ролей — задел под них есть в структуре пользователя)
- Хранилище данных — простой JSON-файл (`data.json`) через lowdb, без внешней БД
- API для заказов и прайса (CRUD)
- Загрузка фото (`/api/upload`), отдача через `/uploads/...`
- Заготовка синхронизации с Kaspi API (модуль `kaspi.js`, до 3 магазинов через переменные окружения) —
  **не проверено на реальном токене**, см. README
- Простой фронтенд-каркас (вход, список заказов/прайса, кнопка синхронизации Kaspi)
- Категории прайса — выпадающий список с возможностью добавить новую "на месте"
